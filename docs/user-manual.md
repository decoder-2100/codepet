## 1. 产品简介

**CodePet** 是一款运行在 Windows 桌面的像素风宠物应用，陪伴程序员工作。它实时监测你的键盘活动，根据编码状态切换动画和姿态；你可以与它聊天、让它分析报错、听它讲段子吐槽，还可以高度定制它的外观和性格。

### 核心特性

- **实时响应** — 根据键盘输入频率自动切换编码/待机/累趴状态
- **LLM 对话** — 支持 6 种 AI Provider（DeepSeek、通义千问、智谱 GLM、百度文心、OpenAI、自定义）
- **Bug 粉碎** — 拖拽报错文本，AI 分析原因并给出修复建议
- **高度自定义** — 6 只预设宠物、40+ 部件组合、4 色调色板、8 种饰品
- **6 种性格** — 幽默、毒舌、温柔、极客、佛系、傲娇
- **3 种主题** — 黑客帝国、复古终端、全息投影

### 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 (x64) |
| WebView2 | 已内置（Win10 20H2+ 自动包含） |
| 内存 | 建议 2GB 以上可用 |
| 磁盘 | 约 50MB（含安装文件） |
| 网络 | 可选，仅 LLM 功能需要 |

## 2. 安装与运行

### 2.1 安装

1. 下载 `CodePet_0.1.2_x64-setup.exe` 安装包
2. 双击运行安装程序，按照向导完成安装
3. 安装完成后 CodePet 将自动启动，出现在屏幕右下角

### 2.2 运行

- **启动**: 安装后自动启动，或从桌面快捷方式/开始菜单启动
- **退出**: 右键点击宠物 → "退出 Quit"，或点击系统托盘图标 → "Quit"

### 2.3 首次启动

首次启动时宠物会出现在屏幕右下角，默认是一只橘猫「橘宝」。此时：

1. 右键宠物 → "设置 Settings" 打开设置窗口
2. 在 **AI 模型** 标签页中填入你的 API Key（推荐 DeepSeek 或通义千问）
3. 点击 **测试连接** 验证配置
4. 点击 **保存** 后开始使用

> **提示**: 不配置 API Key 也能使用大部分功能（动画、音效、离线吐槽等），但聊天和 Bug 分析需要有效的 API Key。

## 4. 宠物交互

### 4.1 键盘感知与状态切换

CodePet 会在后台监听你的键盘活动，每 5 秒计算一次 KPM（每分钟按键数）：

| KPM 范围 | 宠物状态 | 动画 |
|----------|----------|------|
| KPM > 20 | 编码中 | typing（编码节奏） |
| KPM ≤ 20 且持续 30s 无活动 | 待机 | idle（呼吸待机） |
| 连续编码 2 小时 | 累趴 | collapsed（瘫倒呼吸） |

### 4.2 自动隐藏

开启后，当你在 60 秒内没有键盘活动时，宠物窗口会淡出至 8% 透明度（几乎不可见）。恢复打字后宠物会立即重新出现。

> **注意**: 当你打开了设置面板、聊天面板、右键菜单或弹出了气泡时，宠物不会被自动隐藏。

### 4.3 休息提醒

默认每编码 120 分钟（可调整为 30-180 分钟），宠物会弹出气泡提醒你休息，同时播放提示音效。

### 4.4 随机闲时行为

待机状态下，宠物每 8-20 秒会随机执行以下行为之一：

| 行为 | 描述 | 附带气泡 |
|------|------|----------|
| **look-around** | 好奇地左顾右盼 | 30% 概率 |
| **stretch** | 伸懒腰 | 100% 概率 |
| **paw-tap** | 欢快拍爪 | 100% 概率 |
| **happy** | 开心跳跃 | 30% 概率 |

闲时气泡示例：`"摸鱼 ing..."`、`"该喝水了哦！"`、`"好困啊... zzz"`、`"我在看着你哦"` 等共 18 条短语。

### 4.5 窗口拖拽移动

在宠物窗口上按住鼠标左键并拖动超过 5px，会触发窗口拖拽功能，可以改变宠物在屏幕上的位置。宠物窗口默认锚定在屏幕右下角。

**持久化**: 聊天会话同时保存在浏览器 localStorage 和后端文件系统中，关闭应用后不丢失。

### 5.9 流式输出与中断

- 宠物回复逐 token 显示，带有闪烁光标指示器
- 流式输出过程中，输入框和发送按钮被禁用
- 流式完成或出错后自动恢复可输入状态

## 7. AI 模型配置

### 7.1 支持的 Provider

| Provider | 默认模型 | 推荐模型 | 默认 Base URL |
|----------|----------|----------|---------------|
| **DeepSeek** | deepseek-chat | deepseek-chat, deepseek-reasoner | `https://api.deepseek.com/v1` |
| **通义千问** | qwen-max | qwen-max, qwen-plus, qwen-turbo | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| **智谱 GLM** | glm-4-plus | glm-4-plus, glm-4-air, glm-4-flash | `https://open.bigmodel.cn/api/paas/v4` |
| **百度文心** | ernie-4.0 | ernie-4.0 | `https://qianfan.baidubce.com/v2` |
| **OpenAI** | gpt-4o | gpt-4o, gpt-4o-mini, gpt-4.1 | `https://api.openai.com/v1` |
| **自定义** | 空 | 任意 OpenAI 兼容 API | `https://api.example.com/v1` |

### 7.2 配置步骤

1. 打开 **设置 → AI 模型**
2. 选择 **Provider**（下拉框）
3. 填入 **API Key**（密码输入框，不显示明文）
4. **Base URL** 和 **Model** 已自动填入对应 Provider 的默认值，可手动修改
5. 点击 **测试连接** — 成功显示 ✅ 绿色标记，失败显示 🔴 错误信息
6. 点击 **保存** 生效

### 7.3 高级参数（可展开/收起）

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| **Max Tokens** | 256 - 4096 | 1024 | 单次回复最大 token 数 |
| **Temperature** | 0 - 2 | 0.7 | 回复创造性（高）vs 确定性（低） |
| **Top P** | 0 - 1 | 1.0 | 核采样参数 |

### 7.4 流式与非流式

- 默认使用 **SSE 流式输出**，token 逐字显示
- 流式失败时自动回退到 **非流式调用**（整段返回）

## 10. Bug 粉碎机

### 10.1 使用方法

1. 复制报错信息到剪贴板
2. 将文本拖拽到 CodePet 窗口
3. 窗口出现红色虚线边框提示
4. 宠物执行 **crushing** 动画（剧烈震动 → 压扁）
5. 1 秒后展示分析结果气泡，然后切换到 **happy** 动画

### 10.2 AI 分析模式（需要 API Key）

报错文本发送给 LLM，AI 会分析：
- 错误原因
- 修复建议
- 相关代码示例

### 10.3 离线降级模式（无 API Key）

无 API Key 时使用 ASCII 艺术 + 预置吐槽语：

```
╔══════════════════════╗
║   🐛 Bug 粉碎报告      ║
╠══════════════════════╣
```

### 14.3 常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| **宠物不显示** | 自动隐藏已激活 | 开始打字即可恢复，或在设置中关闭自动隐藏 |
| **聊天无回复** | API Key 无效或网络不通 | 检查 API Key，点击"测试连接" |
| **窗口卡住** | WebView2 渲染异常 | 右键 → 退出，重新启动 |
| **Bug 粉碎无结果** | `get_clipboard_text` 返回空 | 确保拖拽时文本在剪贴板中 |
| **启动慢** | 首次加载 WebView2 引擎 | 属正常现象，后续启动应 <3 秒 |
| **退出卡住** | rdev 键盘线程未释放 | 强制关闭即可，已修复大部分场景 |

## 15. 常见问题 FAQ

### Q: 没有 API Key 能用吗？
**A**: 可以。离线功能包括：动画、音效、自动隐藏、休息提醒、随机吐槽/点赞（从预设库）、Bug 粉碎（ASCII 降级模式）。仅聊天和 AI Bug 分析需要 API Key。

### Q: 支持哪些 AI 模型？
**A**: DeepSeek、通义千问、智谱 GLM、百度文心、OpenAI，以及任何兼容 OpenAI API 格式的自定义 Provider。

### Q: 宠物会影响性能吗？
**A**: CodePet 资源占用极低，Canvas 渲染维持 60fps，时间步长上限 50ms 防止性能螺旋。日常使用几乎不影响系统性能。

### Q: 如何备份我的设置？
**A**: 设置保存在 `coderpet_settings.json`（可执行文件同目录），聊天会话保存在 localStorage + 后端文件系统。复制这些文件即可备份。

### Q: 宠物窗口可以移到屏幕其他位置吗？
**A**: 可以。按住宠物窗口拖动即可移动，默认锚定在屏幕右下角。

### Q: 支持 macOS/Linux 吗？
**A**: 当前发布的安装包仅支持 Windows 10/11 (x64)。源代码基于 Tauri v2，理论上支持跨平台编译，但需自行构建。
