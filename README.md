## 快速开始

### 系统要求

- Windows 10+ / macOS 12+ / Linux (需 WebView2)
- 网络连接（用于 LLM 对话和 AI 分析）

### 安装

1. 从 Release 页面下载对应平台的最新版本
2. 解压后直接运行 `coderpet.exe`（无需安装）

### API Key 配置（可选）

如需 AI 对话和 Bug 智能分析功能，需要配置 API Key：

1. 右键宠物 → **设置** → **AI 模型** 标签
2. 选择 Provider（如 DeepSeek、OpenAI 等）
3. 填入 API Key 和模型名称
4. 点击 **测试连接** 验证配置
5. 点击 **保存**

> 不配置 API Key 时，聊天功能会提示"AI 未配置"，Bug 粉碎会使用内置 ASCII 降级方案，核心动画和吐槽功能仍可正常使用。

## 基本操作

### 窗口布局

宠物是一个 **150×180px** 的无边框透明窗口，默认出现在屏幕右下角。始终置顶显示。

### 移动宠物

按住宠物任意位置拖拽，可将其移动到桌面任意位置。

### 右键菜单

在宠物上点击右键弹出菜单：

| 菜单项 | 功能 |
|--------|------|
| 💬 聊一聊 | 打开 AI 聊天面板 |
| 😂 吐个槽 | 宠物随机说一句技术梗吐槽 |
| 🐛 扔个bug | 激活 Bug 粉碎模式（5秒内可拖拽报错） |
| 🎨 自定义宠物 | 打开宠物外观编辑面板 |
| ⚙️ 设置 | 打开设置面板 |

### 系统托盘

系统托盘中有 CodePet 图标：

- **左键单击**：切换宠物窗口显示/隐藏
- **右键菜单**：
  - 显示宠物 — 显示窗口
  - 设置 — 显示窗口并打开设置
  - 退出 — 关闭应用

## 宠物交互

### 宠物状态

宠物根据你的键盘活动自动切换状态：

| 状态 | 触发条件 | 表现 |
|------|---------|------|
| **待机 (idle)** | 停止打字约 30 秒 | 轻微呼吸起伏，偶尔眨眼 |
| **编码中 (typing)** | KPM > 20 | 头部摆动，眼睛微眯 |
| **说话 (talking)** | AI 回复时 | 头部摇晃，嘴巴张合 |
| **开心 (happy)** | Bug 粉碎完成后 | 跳跃，squash & stretch 效果 |
| **粉碎中 (crushing)** | Bug 粉碎时 | 剧烈震动 → 被压扁 → 恢复 |
| **累趴 (collapsed)** | 连续编码 2 小时 | 身体旋转 90 度，压扁，半透明 |

### 聊天气泡

宠物头顶会弹出气泡显示文字，内容包括：
- 吐槽内容（右键 → 吐个槽）
- Bug 粉碎结果
- 休息提醒
- AI 回复（在聊天面板中显示）

气泡在设定时间后自动消失。

### 自动眨眼

宠物每隔 2-5 秒自动眨眼一次，闭眼过程约 40ms，睁眼约 60ms。

## AI 对话

### 支持的 Provider

| Provider | 默认模型 | 备注 |
|----------|---------|------|
| DeepSeek | deepseek-chat | 默认 |
| 通义千问 (Qwen) | qwen-turbo | 阿里云 |
| 智谱 GLM | glm-4 | 智谱 AI |
| 百度文心 | ernie-speed | 百度 |
| OpenAI | gpt-4o | OpenAI |
| 自定义 (OpenAI 兼容) | — | 可配置 base URL |

### 聊天面板

1. 右键 → **聊一聊** 打开聊天面板
2. 在输入框输入消息，按 **Enter** 发送
3. **Shift + Enter** 换行
4. AI 回复会逐 token 流式显示（带闪烁光标）
5. 回复完成后光标消失，消息存入对话历史

### 流式输出

对话采用 SSE 流式输出，回复文字逐 token 出现在聊天面板，无需等待完整回复生成。

### 停止生成

发送新消息或关闭面板会自动停止正在进行的生成。

## Bug 粉碎

### 使用方式

1. 复制一段报错信息到剪贴板
2. 将报错文本从其他窗口**拖拽**到宠物窗口上
3. 宠物区域显示红色虚线边框（拖拽进入时高亮）
4. 释放拖拽后宠物执行粉碎动画

### AI 分析模式（已配置 API Key）

宠物会：
1. 执行粉碎动画（震动 → 压扁 → 恢复）
2. 将报错发送给 LLM 分析
3. 弹出气泡显示分析结果（含吐槽 + 原因 + 修复建议）
4. 切换为开心动画

### 离线降级模式（无 API Key）

- 宠物正常播放粉碎动画
- 弹出 ASCII 艺术风格的粉碎报告
- 显示错误摘要和"重启试试"建议

## 角色系统

打开方式：右键 → **设置** → **宠物角色** 标签

### 宠物名字

自定义宠物名字（默认"橘宝"），会注入到 AI 系统提示词中，对话时宠物会以该名字自称。

### 6 种性格

| 性格 | 回复风格 |
|------|---------|
| 幽默风趣 | 开玩笑，用段子帮用户缓解压力 |
| 毒舌吐槽 | 犀利一针见血，但出发点是为用户好 |
| 温柔治愈 | 暖心鼓励，正能量满满 |
| 技术极客 | 专注技术底层原理和最佳实践 |
| 佛系禅意 | 淡定从容，常用禅意金句 |
| 傲娇 | 嘴上嫌弃但实际很关心 |

### 灵魂档案 (soul.md)

自由编辑 Markdown 格式的宠物身份设定。该内容完整注入到 AI 系统提示词的"## 你的身份"段落中。可以定义：

- 宠物的身份和背景故事
- 行为规则和口头禅
- 知识领域和专业方向

### 技能开关

可多选的宠物能力列表：

| 技能 | 说明 |
|------|------|
| 🐛 Bug 分析 | 分析报错并给出修复建议 |
| 🔍 代码审查 | 审查代码并给出改进意见 |
| 💬 技术闲聊 | 聊技术话题和行业八卦 |
| 🌈 情绪调节 | 在你烦躁时安抚情绪 |
| 😂 吐槽模式 | 用技术梗吐槽产品经理 |
| ⏰ 健康提醒 | 提醒休息、喝水、护眼 |

选中的技能会注入到 AI 系统提示词中，影响对话能力范围。

## 键盘活动监测

### 监测方式

后台进程使用全局键盘钩子监听按键事件，每 5 秒计算一次每分钟按键数（KPM）。

### 状态自动切换

- **编码状态**：KPM > 20 时触发，宠物切换为 typing 动画
- **待机状态**：KPM < 20 持续约 30 秒后切换为 idle 动画
- **连续编码提醒**：连续编码 2 小时后触发 collapsed 状态 + 休息提醒

### 自动隐藏

开启后，长时间无键盘活动时宠物窗口自动隐藏，开始打字后自动恢复显示。

## 常见问题

**Q: 退出后设置会丢失吗？**
A: 不会。所有设置保存在可执行文件同目录下的 `coderpet_settings.json` 中。

**Q: 如何更新 API Key？**
A: 右键 → 设置 → AI 模型 标签，修改后点击保存。

**Q: 宠物窗口消失了怎么办？**
A: 点击系统托盘中的 CodePet 图标，或右键托盘图标选择"显示宠物"。

**Q: 可以不配置 API Key 使用吗？**
A: 可以。核心动画、宠物自定义、吐槽、Bug 粉碎（降级模式）均无需 API Key。

**Q: 如何修改宠物的性格？**
A: 右键 → 设置 → 宠物角色 → 性格下拉框中选择 → 保存。

**Q: 自定义的外观会保存吗？**
A: 自定义面板中点击"应用"后生效，如需持久化请在设置面板中保存。

**Q: CPU 占用高吗？**
A: 空闲时 CPU 占用 < 5%，Canvas 渲染帧率 60fps。

## 技术信息

### 技术栈

| 层 | 技术 |
|----|------|
| 前端框架 | React 19 + TypeScript |
| 构建工具 | Vite 6 |
| 状态管理 | Zustand 5 |
| Canvas 渲染 | 自研动画引擎（关键帧插值 + 过渡混合） |
| 桌面框架 | Tauri v2 |
| 后端语言 | Rust |
| LLM 调用 | OpenAI 兼容 API 格式（SSE 流式） |
| 键盘监测 | rdev 全局钩子 |
| 音效 | Web Audio API |

### 窗口属性

- 尺寸：150 × 180px（固定）
- 背景：透明
- 边框：无边框（Windows 使用 DwmSetWindowAttribute 禁用 NC 渲染）
- 置顶：始终置顶
- 任务栏：不显示

### 数据文件

设置文件：`{可执行文件目录}/coderpet_settings.json`

### 构建

```bash
# 开发模式
npm run tauri dev

# 生产构建
npm run tauri build
```
